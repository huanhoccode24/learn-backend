TỔNG HỢP KIẾN TRÚC & CHỨC NĂNG AUTHENTICATION

Báo cáo hệ thống với Next.js 15, Auth.js (v5) & PostgreSQL

1. Cấu hình Cốt lõi Next-Auth & Edge Middleware

Kiến trúc tách làm 2 file (auth.config.ts và auth.ts) nhằm đảm bảo Middleware của Next.js (chạy trên Edge Runtime) không bị báo lỗi khi import các thư viện Node.js nguyên thủy (như bcryptjs, pg).

Các bước thực hiện:

  • Tạo auth.config.ts chỉ chứa Providers trống và Callbacks, xuất ra Edge middleware.
  • Tạo auth.ts kết nối Adapter @auth/pg-adapter và khởi tạo Providers thật sự (Credentials, Github, Google).
  • Tạo thư mục API cho Next-Auth: src/app/api/auth/[...nextauth]/route.ts

Ví dụ Code - Phân lớp Middleware bằng Callbacks (auth.config.ts):


export const authConfig = {
  pages: { signIn: '/login' },
  session: { strategy: 'jwt' },
  callbacks: {
    authorized({ auth, request }) {
      const isLoggedIn = !!auth?.user;
      const isOnDashboard = request.nextUrl.pathname.startsWith('/dashboard');

      // Chặn đường dẫn dashboard nếu chưa đăng nhập
      if (!isLoggedIn && isOnDashboard) return false; 
      return true;
    }
  },
  providers: [] // Khai báo trống cho Edge
};

2. Đăng ký & Mã hoá Mật khẩu (Register)

Cho phép người dùng tạo tài khoản. Mật khẩu được mã hóa an toàn trước khi lưu vào PostgreSQL.

Các bước thực hiện & Thư viện sử dụng:

  • Thư viện: zod (Validation form), bcryptjs (Hashing mật khẩu), pg (Query DB).
  • Bước 1: Zod kiểm tra dữ liệu đầu vào.
  • Bước 2: Query kiểm tra trùng email (SQL thô: SELECT id FROM users).
  • Bước 3: Mã hoá pass: await bcrypt.hash(password, 10).
  • Bước 4: Insert vào PostgreSQL và trả về kết quả cho phía Client hiển thị.

Ví dụ Code - Server Action Đăng ký (actions/auth/register.ts):


import bcrypt from 'bcryptjs';
import pool from '@/lib/db';

export const register = async (values) => {
  // 1. Phân bổ và validate
  const { email, password, name } = values;

  // 2. Hash Password
  const hashedPassword = await bcrypt.hash(password, 10);

  // 3. Lưu vào Database
  await pool.query(
    'INSERT INTO users (name, email, password, role) VALUES ($1, $2, $3, $4)',
    [name, email, hashedPassword, 'USER']
  );

  return { success: 'Đăng ký tài khoản thành công!' };
};

3. Đăng nhập Auth.js (Login)

Các bước thực hiện:

  • Tích hợp Provider: Khai báo thẻ Credentials trong mảng providers tại auth.ts.
  • Khớp dữ liệu: Query user qua email hiện có, sử dụng bcrypt.compare để xử lý giải mã và đối chiếu mật khẩu.
  • Kích hoạt Server Action: Gọi hàm signIn('credentials', { email, pass }) tại UI Client.
  • Xử lý NextAuth Error: Nếu sai thì Catch lấy AuthError -> CredentialsSignin. Chú ý luôn throw error nếu redirect.

Ví dụ Code - Hàm gọi Login Server Action (actions/auth/login.ts):


import { signIn } from '@/auth';
import { AuthError } from 'next-auth';

export const login = async (values) => {
  try {
    // Kích hoạt Middleware NextAuth auth.ts đăng nhập
    await signIn('credentials', {
      email,
      password,
      redirectTo: '/dashboard', 
    });
  } catch (error) {
    if (error instanceof AuthError) {
      if (error.type === 'CredentialsSignin') return { error: 'Sai mật khẩu!' }
    }
    throw error; // Quan trọng để module Redirect Next.js chạy tiếp
  }
};

4. Bảo mật Hai Lớp (2FA OTP) qua Email

Logic bảo mật cao nhất: Bắt buộc user (bất kể dùng mật khẩu hay OAuth) phải nhận được mã OTP trên Email rồi mới cho vào xem dashboard.

Các bước phát triển:

  • Tạo mã Tự động: Bắt sự kiện signIn bên trong auth.ts. Cứ có người đăng nhập đúng mật khẩu => Dùng thư viện crypto sinh ngẫu nhiên số 6 chữ số, thời hạn 5 phút, lưu vào DB bảng two_factor_tokens (xóa mã cũ của email đó tránh rác).
  • Gửi Email (Nodemailer): Lấy số đó đẩy qua SMTP Gmail transporter.sendMail gửi cho user (chỉnh HTML mail cho đẹp).
  • Cờ Session (Callback JWT): Đánh dấu property token.isTwoFactorVerified = false ở session ban đầu. Middleware Next.js đọc giá trị này và redirect user về trang nhập /verify-otp.
  • Xác thực phía Server: Nhận mã người dùng nhập, query database. Kiểm tra trùng khớp và thời hạn (expires < Date.now).
  • Mở Khóa: Nếu đúng, xóa record khỏi DB. Thiết lập Cookie 2fa_verified bằng next/headers cookies().set() nhằm override bug Session bên Next-Auth => Middleware thông qua.

Bắt sự kiện (auth.ts):


callbacks: {
  async signIn({ user }) {
    if (user.email) {
      // Logic gửi 2FA OTP tự động
      const tokenObj = await generateTwoFactorToken(user.email);
      await sendTwoFactorTokenEmail(tokenObj.email, tokenObj.token);
    }
    return true; // Cho phép tạo session 
  },
}

Sinh Token 6 số ngẫu nhiên:


import crypto from 'crypto';

export const generateTwoFactorToken = async (email) => {
  // random từ 100000 => 999999
  const token = crypto.randomInt(100000, 1000000).toString(); 
  const expires = new Date(new Date().getTime() + 5 * 60 * 1000); // 5 phút

  // Xoá rác + insert record mới xuống PosgeSQL ...
  return { email, token, expires };
};

Mở rào tại Middleware khi OTP đúng (auth.config.ts):


import { cookies } from 'next/headers';

authorized({ auth, request }) {
  // Kiểm tra cờ JWT
  let isVerifiedOTP = auth?.user?.isTwoFactorVerified === true;

  // FIX BUG NextAuth bằng Cookie tự tạo khi giải mã Server Action
  if (request.cookies.get('2fa_verified')?.value === auth?.user?.email) {
     isVerifiedOTP = true;
  }

  // Khóa cổng
  if (isLoggedIn && !isVerifiedOTP && !isOnVerifyPage) {
     return Response.redirect('/verify-otp');
  }
}
